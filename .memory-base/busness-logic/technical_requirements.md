# Telegram-бот для предварительного заказа обедов в офисе

## Роли пользователей

| Роль | Описание |
|------|----------|
| **Сотрудник** | Делает заказ себе (возможно, и нескольким людям) |
| **Офис-менеджер** | Загружает меню, получает ежедневные отчеты, общается с кафе |

---

## Базовый сценарий сотрудника

1. Сотрудник открывает Telegram-бота
2. Выбирает офис / кафе
3. Бот показывает меню подключенных кафе (или выбранного кафе)
4. Сотрудник выбирает блюда
5. Подтверждает заказ → получает резюме и, при желании, цену
6. Офис-менеджеру к определённому времени (дедлайн заказа) бот отправляет готовый отчёт

---

## Обязательные функциональные требования

### Для сотрудника:
- [ ] Авторизация в боте по Telegram-ID
- [ ] Просмотр меню
- [ ] Оформление заказа
- [ ] Управление заказом

### Для менеджера:
- [ ] Загрузка сотрудников (Telegram-ID, имя, офис)
- [ ] Управление сроками, меню, доступными кафе, днями, когда доступен заказ
- [ ] Просмотр заказов
- [ ] Получение агрегированного отчёта по кафе
- [ ] Запрет изменений заказов после дедлайна

---

## Необязательные функциональные требования

Дополнительные возможности, которые сделают проект полезнее и/или интереснее:

- [ ] Личный баланс/статистика сотрудника
- [ ] Возможность выгрузки в различных форматах / отправка на почту
- [ ] Возможность отправки заказов напрямую в кафе и рестораны
- [ ] Умные подсказки и рекомендации
